# Temporal Displacement

Inhabit a different time horizon. Most decisions are trapped in the present — this mode breaks temporal fixedness.

## Cognitive Move

Leave the present. Inhabit a future (or past) vantage and ask: "What's obvious from here that isn't visible from now?" Not extrapolation — genuine displacement. Reason FROM the future, not about it.

## Process

1. **Name the default horizon.** "This decision is framed for [weeks/quarter/sprint]."
2. **Choose displacement:** `1y` (what will you wish you'd done?), `10y` (what's obviously true?), `100y` (what's durable vs. transient?), `retrospective` (what changed since 5 years ago that you're not accounting for?). User can specify: `/unleash temporal 10y`.
3. **Inhabit fully.** Write as: "It's 2036. Looking back at this decision..."
4. **Identify what's obvious from there.** Trends that compound, decisions that lock in, options that expire, assumptions that age badly.
5. **Bring insight back.** What should you do NOW given what's obvious from the displaced horizon?

## Quality Gate

PASS: Specific insight invisible from present but obvious from displaced horizon, insight changed the present decision, displaced reasoning was problem-specific.
FAIL: Today's trends extrapolated linearly, temporal displacement announced but present reasoning happened, generic futurism.

If output quality is unclear, read `./examples.md` (temporal section) for calibration.

Chains: first-principles, reframe. Use shortest horizon that produces insight.
