# Reframe / Blue Ocean

Stop competing within current rules. Ask if you're playing the right game — then change the game to one where you win by default.

## Cognitive Move

Treat the current problem framing as the real problem. Not "how do we win?" but "why are we competing here?" Find the game where your strengths are decisive and weaknesses irrelevant.

## Process

1. **Name the game.** "We're playing the 'build a better X' game." State rules, incumbents, scoring system.
2. **Identify what the game rewards.** Feature parity? Price? Brand? Distribution? The scoring system favors incumbents.
3. **Ask: "What if this game is wrong?"** What would you build if this category didn't exist?
4. **Four Actions Framework:**
   - **Eliminate**: Industry factors you can drop entirely
   - **Reduce**: Factors to reduce below industry standard
   - **Raise**: Factors to raise above standard
   - **Create**: Factors the industry has never offered
5. **Define the new game.** What category? Who wins? Why you? Direct comparison with incumbents should feel like comparing apples to orchestras.

## Quality Gate

PASS: Current game named specifically, genuine alternative emerged (not a niche), new game leverages real strengths, ≥1 counterintuitive eliminate/create.
FAIL: Just positioning language for same product, new game requires capabilities you don't have, nothing was eliminated.

If output quality is unclear, read `./examples.md` (reframe section) for calibration.

Chains: empathy, scale-break. Tension: constraint embrace (productive — "given our constraints, what game would we dominate?").
