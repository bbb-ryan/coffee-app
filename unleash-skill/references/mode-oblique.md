# Oblique Approach

Introduce randomness to escape stuck patterns. When direct approaches fail, go sideways. Eno: "Honor thy error as a hidden intention."

## Cognitive Move

Abandon the direct path. Introduce a random constraint or provocation and force a connection to the problem. Creative blocks come from over-specifying the goal. Random perturbation knocks you out of the local optimum.

## Process

1. **Name the stuck pattern.** What approach keeps being tried? What groove are you in?
2. **Pick an oblique provocation.** Choose the most ABSURD one for this problem, not the most relevant:
   - "What would this look like if it were easy?"
   - "Honor thy error as a hidden intention."
   - "Most embarrassing version that would still work?"
   - "What if you had to launch tomorrow?"
   - "Do the opposite of the category leader."
   - "Remove the most important feature. What's left?"
   - "What if the constraint was the product?"
   - "What would a 12-year-old do?"
   - "Use an unacceptable [color/technology/approach]."
   - "Emphasize the flaws."
   - "What would make this funny?"
   - "Reverse the order of everything."
   - "What if this were a one-time event, not a product?"
   - "Give it away. Now what?"
3. **Force the connection.** Take it seriously for 60 seconds. Don't dismiss.
4. **Extract the useful fragment.** The provocation surfaces something — a direction, a constraint, a reconsideration.
5. **Integrate or honestly discard.** Not every attempt produces gold.

## Quality Gate

PASS: Provocation was genuinely oblique, connection forced and explored, something shifted even if final approach doesn't use the provocation.
FAIL: "Oblique" was conventional brainstorming, provocation chosen because it already fit, listed but not explored.

If output quality is unclear, read `./examples.md` (oblique section) for calibration.

Chains: taste (generate weird → select right), constraint (provocation becomes constraint). Conflicts with analytical modes — that's the point.
