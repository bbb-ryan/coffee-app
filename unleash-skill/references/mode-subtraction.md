# Subtraction

Remove elements until only the essential remains. Default action = REMOVE, not keep. Not minimalism as aesthetic — systematic elimination that reveals what actually matters.

## Cognitive Move

Every element must justify its existence. For each feature, step, component, dependency: "Does removing this make the whole worse?" Nothing happens → remove. Gets harder but works → scrutinize. Whole thing breaks → essential.

## Process

1. **List every element.** Be granular: each service, endpoint, step, dependency, role.
2. **Remove-test each one.** What happens without it? Nothing / harder / breaks?
3. **Remove everything non-essential.** Be aggressive. You can add back.
4. **Examine what remains.** The skeleton reveals true structure — often simpler and more powerful.
5. **Check for double duty.** Can remaining elements serve multiple purposes?

## Quality Gate

PASS: ≥1 element removed that felt important but was optional, stripped version is more coherent (not just smaller), removal revealed hidden priority.
FAIL: Listed removals without committing, cosmetic subtraction (fewer words) not structural (fewer features/steps), same solution described shorter.

If output quality is unclear, read `./examples.md` (subtraction section) for calibration.

Chains: taste (strip, then make bones beautiful), first-principles. Tension: scale-break (productive).
