# Systems / Leverage

See the whole system. Find where small intervention → disproportionate effect. Most effort goes to low-leverage points.

## Cognitive Move

Stop seeing the problem as a thing — see it as a system with feedback loops, delays, bottlenecks. Find the intervention that's 10% effort, 80% result. Meadows' hierarchy: most powerful interventions change rules or paradigm, not parameters.

## Process

1. **Map the system.** Key elements and connections. "A drives B, B constrains C, C feeds A." Look for:
   - Feedback loops (reinforcing + balancing)
   - Delays (cause-effect time lags — where most interventions fail)
   - Bottlenecks (narrowest pipe constraining flow)
   - Stocks (what accumulates: users, trust, debt, knowledge)
2. **Identify current intervention point.** Usually it's parameters (lowest leverage).
3. **Walk up the leverage hierarchy:** Parameters → Buffers → Flow structure → Rules → Information flows → Goals → Paradigm.
4. **Find highest-leverage intervention you can actually execute.** Not theoretical highest — available highest.
5. **Articulate the cascade.** "Change X → shifts Y → compounds into Z." Make chain explicit.

## Quality Gate

PASS: Feedback loop or systemic dynamic identified outside original framing, proposed intervention at higher leverage than current approach, cascade articulated.
FAIL: "Systems thinking" invoked but output is linear plan, leverage point is vague ("change culture"), no loops/delays/bottlenecks identified.

If output quality is unclear, read `./examples.md` (leverage section) for calibration.

Chains: inversion, reframe. Can produce analysis paralysis — set scope boundary.
