How skills load into Claude's memory

A skill isn't always "on." Claude Code uses a three-tier loading pattern so your conversation doesn't pay for context it isn't using. Understanding this helps you (and us) design skills that stay cheap and fast.

Tier 01
Description

Short paragraph at the top of every skill — describes the role and the keywords that trigger it.

Always loaded
Tier 02
SKILL.md body

The main instructions for the role — rules, process, gotchas, output format.

Loaded on invocation
Tier 03
references/ files

Deep detail — voice guides, illustration rules, long checklists. Loaded only when the skill explicitly reads them.

Skill 01

The copywriter

Every word a user reads — warm, inviting, never intimidating.

Triggers on keywords
copy wording headline CTA empty state error message flavor description tone voice microcopy "make this sound better"
Before writing
  • Read references/brand-voice.md — tone, jargon rules, do/don'ts, page voice notes, myth-busting, depth moments
  • Match jargon level to page depth — surface pages use plain language, deep pages use SCA vocabulary
  • For flavor descriptions, consult src/data/flavor_taxonomy.json for SCA wheel vocabulary, then translate to everyday anchors
What it owns
  • Headlines & hero — sensory, curious, not generic
  • CTAs, labels, microcopy — 2–4 words, specific
  • Flavor descriptions — sensory anchor first, SCA note after
  • Empty states & errors — always paired with a next step
  • Myth-busting copy (wedge content) — co-owned with designer
  • Depth moments — co-owned with designer + researcher
Gotchas · seven project-specific traps
"Discover" / "Explore" in hero headlines.Claude defaults to these; they're generic SaaS voice.
Title Case headings.We use sentence case everywhere. Claude sometimes Title-Cases despite the rule.
"No results found."Claude's default empty state. Rewrite to active, invitational, with a CTA.
Em-dash drift.Claude favors em dashes in warm copy. One per paragraph max.
Marketing fluff."Elevate your coffee experience," "craft your journey" — strip on sight.
Over-long CTAs.Claude drifts to 5+ words. Cap at 2–4.
Gendered or equipment-assuming copy."Grab your V60" assumes gear. Neutral wins.

Skill 02

The designer

Visual decisions inside our Tailwind system — tokens, layouts, illustrations.

Triggers on keywords
design layout component UI responsive mobile card grid spacing color Tailwind "make this look better"
Before designing
  • Read src/app/globals.css for current theme tokens and custom utilities
  • Scan src/components/ for existing patterns — reuse before inventing
  • For illustrated elements, read references/illustration-system.md — Bryan's house style
Decisions
  • Check what exists. Don't make a second card pattern.
  • Stay in tokens. No hardcoded hex, no arbitrary Tailwind colors.
  • Tailwind-only. No component-level CSS files.
  • Show reasoning briefly so the team stays aligned.
  • Flag copy — hand to the copywriter.
Design tokens · the palette in use
  • cream / cream-dark — page + card backgrounds
  • espresso / espresso-light — primary text, dark UI
  • roast / roast-light — secondary text, labels, link color
  • caramel / caramel-light — accents, highlights, hover states
  • sage / sage-light — success states, high-score indicators
  • Typography: Inter (sans) + Georgia (serif). Moving toward Fraunces per CONCEPT-V2 — update the skill once globals.css is changed.
  • No grays. Use cream-dark, roast-light, or espresso-light.
Gotchas · seven palette & system traps
bg-gray-* / text-gray-*.Our palette has no grays. Use cream-dark, roast, roast-light, espresso-light.
Generic blue links or buttons.No blue in UI. Links are roast; primary CTAs use espresso or caramel.
#93C5FD (cool blue) in UI.Reserved for water/steam in illustrations only. Never a button color.
outline-none on focus.Hard a11y rule — always add a visible replacement (ring or custom outline).
Font swap creep.Don't introduce new font families. Inter + Georgia is current.
Touch targets under 44px.Accessibility regression. Use min-h-[44px] or padding.
Custom CSS files.No component-level .css. All styling is Tailwind utilities.

Skill 03

The researcher

Coffee domain expertise + product research, grounded in our data.

Triggers on keywords
coffee origin process washed natural SCA Q-grader flavor wheel competitor data analysis beans dataset
Ground in the data
  • src/data/beans.json — 1,338 beans from CQI cupping data
  • src/data/beans_curated.json — 20 hand-picked Phase 1 beans
  • src/data/flavor_taxonomy.json — SCA Flavor Wheel
  • src/data/schema.md — data model
  • Rule: read the JSON before claiming a count or pattern. Show your work.
What it owns
  • Coffee domain Q&A — origins, processing, SCA protocol, flavor science
  • Dataset analysis — distributions, gaps, correlations in beans.json
  • Competitive research — Angel's Cup, Beanhunter, Roast.World, Vivino, Untappd (adjacent)
  • Depth moment accuracy — verify coffee is right before anything ships
Gotchas · six domain traps
"Washed" ≠ "clean."Washed is a processing method. Clean is a cupping descriptor (Clean Cup). Don't substitute.
CQI vs SCA.Our data is CQI cupping. SCA informs scoring, but they're distinct institutions.
Altitude ↔ acidity claims.A real correlation in specialty coffee — but verify it holds in OUR data before asserting.
"Specialty" threshold.80+ is the SCA line. Below 80 exists in our data — don't imply everything is specialty.
Natural ≠ fruity.Often are, but not a rule. Ethiopian natural ≠ every natural. Check the notes on the bean.
Generalizing from 20.beans_curated.json ≠ the 1,338. Be explicit about which set you analyzed.
References
— No references folder yet. Skill is small enough to stay single-file. Add one if domain guidance grows (e.g., processing-methods.md, cupping-protocol.md).

Skill 04

The QA reviewer

Quality gate before anything ships — conventions, a11y, consistency, tone.

Triggers on keywords
review audit check QA accessibility a11y responsive convention TypeScript regression "does this follow our conventions"
Before reviewing
  • Re-read CLAUDE.md — source of truth for conventions
  • Stack: Next.js 15 App Router · Tailwind v4 · TypeScript · JSON data · no database
  • Pull references/checklist.md — treat it as a menu, not a script
How it works
  • Identify what changed (new component? data shape? copy only?)
  • Pull relevant checklist sections — not all of them
  • Report by severity: Must fix · Should fix · Nice to have
  • Hand off — design → designer, copy → copywriter, coffee → researcher
  • Always run npx tsc --noEmit and npm run build
Gotchas · nine high-signal traps
outline-none without replacement.Always a Must Fix a11y violation.
any types.Should be rare. Usually Bean from @/lib/beans fits.
Images without meaningful alt.Decorative needs alt=""; meaningful needs descriptive alt.
Empty states with no CTA.Both a copy and UX issue — route to copywriter.
next dev passes, next build fails.Always run build — SSR boundary errors only show there.
Hardcoded hex / bg-gray-* / blue-500.Palette violations — hand to designer.
Raw localStorage.Must use useLocalStorage hook. Raw breaks SSR.
Diary state outside DiaryProvider.Always through the context.
Tone drift.If copy feels off-brand, route to copywriter — don't rewrite yourself.
References · lazy-loaded
checklist.md Full review menu — code conventions · design consistency · a11y · mobile · copy & tone · performance & data. Pull what's relevant; don't walk every row.

Extending the system

Adding a gotcha — every time Claude trips on something new (says "Discover" again, introduces a fresh hex color), open the right SKILL.md and add a line to the Gotchas section. This is the highest-signal content a skill can hold.

New bulk content — if a section in SKILL.md is growing past ~30 lines, split it into references/some-topic.md and leave a one-line pointer. Keeps the loaded context lean.

Committing to git — .claude/skills/ is project-scoped. Commit it so every teammate benefits from the same roles.

Adding a fifth skill — resist unless the role is genuinely distinct. Overlapping skills dilute trigger accuracy. Four is the right shape for now.
